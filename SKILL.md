# Marketing Agent — WPP Open × CoreAI 混合架构

## Overview

Marketing Agent 是一个综合营销知识 + 策略引擎 skill。架构哲学融合两大范式：

- **WPP Open 模式**：模块化的营销操作系统——十个能力模块 + 内容生成 + 输出模板，按需调用
- **CoreAI 模式**：数据驱动的 AI 决策引擎——所有分析基于可追溯的逻辑链，不做无依据的判断
- **前置路由**：7 种工作模式对应 7 类问题，深度问题用 Peiyu 协议（营销版），速度问题用快速框架，执行问题用模板

当前版本是「纯知识层」——包含体系化的营销知识和方法论。后续迭代接入垂直行业数据（酒水供应链），升级为「知识 + 数据双引擎」。

## When to Use

- 分析品牌定位、品牌架构、品牌资产（含全新品类的品类创造策略）
- 选择适用的营销方法论或分析模型
- 规划营销战役（Campaign）或全年营销策略
- 分析线下门店策略、终端陈列、终端动销、门店分级
- 策划线下活动——品鉴会、品牌之夜、经销商大会、快闪店
- 评估广告投放渠道、投流策略、预算分配
- 热点事件匹配——判断一个热点是否值得追、怎么追
- 撰写营销内容——广告文案、社媒帖子、活动邀请函、策略文档
- 解释营销专业术语或模型
- 对营销方案做批判性评估

**Don't use for:**
- 纯数据查询（如「酒水行业2024年市场规模」）→ 需要结合数据层
- 法律合规审查 → 需结合法务
- 预算审批、合同签署 → 涉及金额的决策
- 危机公关实时回应 → 高风险需人类判断
- 具体平台的投放操作教程（如「巨量引擎怎么建计划」）

## 产品定义

### 用户层级

| 用户 | 身份 | 使用场景 | 对 Agent 的要求 |
|:--:|------|------|------|
| **L1 咨询顾问** | 你自己 | 加速方案产出 | 策略框架 + 数据支撑 + 内容快速生成 |
| **L2 品牌方市场总监** | 有营销知识 | 微信问策 | 引导式交互 + 决策建议 + 可解释性 + 方案文档 |
| **L3 经销商** | 营销知识有限 | 「我的店怎么搞」 | 执行清单 + 算账逻辑 + 极简语言（延后开发） |

当前版本聚焦 L1 + L2。

### 决策边界（红线）

**绝对不做**：
- 预算审批、合同签署建议
- 危机公关的实时回应建议
- 品牌价值观判断（AI 无法理解文化微妙性）
- 法律合规建议

**做但标注置信度**：
- 基于不完全数据的策略推演 → 标注假设
- 竞品策略推测 → 标注「推测」
- 趋势预判 → 标注时态

**降级策略**：当信息不足时，Agent 输出分析框架 + 要求补充信息，不硬给确定答案。

### 知识保鲜策略

| 知识类型 | 更新方式 | 频率 | 负责人 |
|----------|----------|:--:|:--:|
| 营销理论/模型 | 基本不变，无需主动更新 | — | — |
| 平台机制（抖音/腾讯/小红书） | 人工触发审查 + 行业媒体监控 | 每季度 | zoe 审查 |

## 利益相关方视角（所有模块输出的横向约束）

营销和 PR 不是对「消费者」一个人说话。每个输出必须回答：**各利益相关方分别看到什么？**

### 利益相关方地图

```
外部                                    内部
├── 消费者（最终用户）                   ├── 品牌方管理层
├── 经销商/终端老板                      ├── 销售团队
├── 行业媒体/酒评人/侍酒师               ├── 投资人/董事会
├── 大众媒体/生活方式媒体                ├── 员工
├── KOL/KOC                             └── 供应链/合作伙伴
├── 行业协会（酒协等）
├── 政府/监管部门
└── 社区/公众
```

### 沟通矩阵（每个模块产出时默认检查）

| 利益相关方 | 关心什么 | 希望从你这里听到什么 | 通过什么渠道触达 |
|------------|----------|-------------------|-----------------|
| 消费者 | 产品体验、价格、在哪买 | 「这对你有什么好处」 | 社媒/广告/终端/KOL |
| 经销商 | 利润、周转、品牌支持 | 「为什么卖这个能赚钱」 | 经销商大会/业务代表/政策文件 |
| 媒体/酒评人 | 新闻价值、专业信息 | 「为什么值得写一篇」 | 新闻稿/样品/独家信息/Pitch |
| 监管/协会 | 合规、行业责任 | 「我们是负责任的品牌」 | 正式公文/协会会议/ESG报告 |
| KOL/KOC | 内容素材、独特体验 | 「你的粉丝会喜欢这个」 | 定制体验/样品/活动邀请 |
| 投资者/管理层 | 增长、ROI、风险 | 「我们的策略和结果」 | 月报/季报/董事会材料 |
| 员工/销售团队 | 自豪感、清晰的指令 | 「为什么在这里工作/卖这个值得」 | 内部通讯/培训/激励方案 |

### 强制规则

任何模块输出策略或内容时，至少标注**三个利益相关方的视角**：
- 消费者看到什么 → 必须
- 经销商/渠道看到什么 → 酒水行业必须
- 至少一个其他利益相关方 → 按场景选

### 沟通基调一致性检查

品牌对外沟通时，不同渠道、不同受众，语调可以不同，但**核心信息必须一致**。不一致的信号会造成品牌信任折扣。

```
✓ 正确：「同一款酒，对消费者说好喝，对经销商说好卖，对媒体说好故事」
✗ 错误：「对消费者说纯天然，对经销商说成本低，对媒体说高科技」
   ← 三个版本的信息互斥，迟早穿帮
```

### Module 1: Brand Concepts 品牌概念体系

品牌是营销的根基。本模块覆盖从品牌本质到品牌激活的全链路概念。

**快速索引**——需要以下内容时加载 `references/brand-concepts.md`：
- 品牌定义（AMA 定义、品牌金字塔、品牌-产品光谱）
- 品牌识别系统（Kapferer 棱镜、Aaker 模型、品牌原型 12 种）
- 品牌定位（STP → 定位陈述公式、重新定位策略、品牌架构 4 型）
- 品牌资产（Aaker 五星模型、Keller CBBE 金字塔、品牌资产评估方法）
- 品牌激活（年轻化 vs 复兴、品牌延伸评估）
- **定价策略**（附录 C）— 价格带选择、渠道价差设计、涨价降价策略、区域管理、线上线下冲突
- **品牌级场景锁定与教育策略**（附录 A）— 场景锁定公式、品牌版 4 层教育节奏、种子用户选择。与 Module 7 品类创造同源工具，降维应用于已有品类
- **高管可见度与思想领导力**（附录 B）— 创始人/酿酒师个人 IP 策略、采访准备
- **品牌使命、愿景、价值观**（附录 D）— 品牌为什么存在、去哪、怎么做事
- **品牌语调**（附录 E）— Tone of Voice 四维模型 + 一致性铁律

**实战速查**：

**品牌定位一句话公式**：
```
对于 [目标人群]，[品牌名] 是 [品类] 中
能够提供 [核心差异化利益] 的 [品牌]
因为 [RTB——相信的理由]。
```

**品牌架构 4 型决策表**：

| 架构类型 | 适用条件 | 案例 |
|----------|----------|------|
| Branded House（统一品牌） | 产品关联度高、目标一致 | Apple、Nike |
| House of Brands（独立品牌） | 目标人群/定位差异大 | P&G、百威英博 |
| Endorsed Brand（背书品牌） | 子品牌需要母品牌信用 | 万豪 × Courtyard |
| Hybrid（混合型） | 收购整合期或市场差异大 | 可口可乐（部分独立部分背书） |

### Module 2: Marketing Methods & Models 营销方法论与模型

营销方法论按用途分为四大类。详细内容见 `references/marketing-models.md`。

**A. 战略分析模型**：

| 模型 | 用途 | 适用场景 |
|------|------|----------|
| STP | 细分→目标→定位，三层推导 | 新品牌入市、品牌重塑 |
| SWOT | 内外部分析 | 年度策略、竞品分析 |
| PESTEL | 宏观环境扫描 | 行业进入评估 |
| Porter 5 Forces | 行业竞争结构 | 投资/进入决策 |
| BCG Matrix | 产品组合管理 | 多产品线资源分配 |
| Ansoff Matrix | 增长方向选择 | 市场/产品扩展决策 |

**B. 消费者/用户模型**：

| 模型 | 核心逻辑 | 适用场景 |
|------|----------|----------|
| AIDA | Attention → Interest → Desire → Action | 传统广告效果评估 |
| 消费者决策旅程 (5A) | Aware → Appeal → Ask → Act → Advocate | 全链路消费者运营 |
| Jobs to be Done | 用户「雇佣」产品完成什么任务 | 产品创新、定位 |
| RFM | Recency × Frequency × Monetary | CRM、用户分层 |

**C. 品牌传播模型**：

| 模型 | 核心逻辑 | 适用场景 |
|------|----------|----------|
| DAGMAR | 定义广告目标以衡量广告效果 | 广告战役 KPI 设定 |
| FCB Grid | 高/低卷入 × 理性/感性，4 象限 | 创意策略方向 |
| 6Ms | Mission, Market, Money, Message, Media, Measurement | 系统性传播规划 |

**D. 数字化/增长模型**：

| 模型 | 核心逻辑 |
|------|----------|
| AARRR (Pirate Metrics) | Acquisition → Activation → Retention → Revenue → Referral |
| RACE | Reach → Act → Convert → Engage |
| 增长飞轮 | 获客→激活→留存→变现→推荐，循环加速 |
| 竞争动态应对 | L1-L4 威胁分级 × 响应矩阵 + 竞争情报体系 |
| 品牌组合资源分配 | BCG 升级：明星/金牛/问题儿童/瘦狗的实际预算分配逻辑 |
| 度量与归因 | 最后点击/线性/MMM + Geo-test + 品牌增长归因框架 |

**模型选择决策树**：

```
问题类型？
├─ 「我们是谁/该往哪走」→ STP + Brand Positioning
├─ 「消费者怎么想/怎么买」→ CDJ(5A) + JTBD
├─ 「怎么传播/怎么投」→ 6Ms + DAGMAR + FCB Grid
├─ 「产品组合怎么管」→ BCG Matrix + Ansoff
├─ 「增长怎么做」→ AARRR + 增长飞轮
├─ 「竞争对手很强」→ Porter 5F + SWOT
└─ 「行业还能不能进」→ PESTEL + Porter 5F
```

### Module 3: External Events Management 外部事件管理：热点与危机

热点是找机会，危机是防风险——同一枚硬币的两面。热点部分详见 `references/trending-events.md`；危机预备部分见该文件中「危机预备框架」章节。

**热点生命周期**：

```
潜伏期(2-6h) → 爆发期(6-24h) → 峰值期(24-48h) → 衰退期(48-72h) → 长尾期(3-7d)
     ↑               ↑                ↑               ↑
  最早介入窗口    最佳追热点窗口   参与仍然有效     价值递减
```

**热点-品牌匹配度评估矩阵（热点营销最重要的工具）**：

对任何热点，从 5 个维度打分（1-5）：

| 维度 | 评估内容 | 权重 |
|------|----------|:--:|
| 相关性 | 热点话题和品牌/品类有无自然关联？ | 30% |
| 调性匹配 | 热点的情绪基调（幽默/感动/愤怒/震惊）是否匹配品牌人设？ | 25% |
| 风险度 | 追这个热点翻车的概率？涉及敏感话题？可能被解读为消费灾难？ | 25% |
| 时效窗口 | 还剩多少时间有效参与？ | 10% |
| 传播势能 | 热点的扩散速度和广度如何？ | 10% |

**决策规则**：
- 综合分 ≥ 4.0 → 全力追
- 3.0-3.9 → 选择性参与
- 2.0-2.9 → 保持观望
- < 2.0 → 放弃

**热点「绝对不碰」红线**：
- 涉及人员伤亡的灾难事件
- 政治敏感事件
- 与竞品绑定的话题（避免为他人做嫁衣）
- 品牌核心价值观冲突的话题（如高端品牌追低俗梗）

### Module 4: Media & Communications 媒介与传播

覆盖 paid / earned / owned 三种媒介逻辑。本文件覆盖**跨平台通用的投流策略**（竞价机制、出价策略、渠道组合、AB 测试）。具体平台操作（巨量引擎/腾讯/小红书后台配置）由对应平台 skill 承载——详见 `references/ad-placement.md` 末尾的「平台执行层分流」。

**投流渠道全景**：

```
                    投流渠道
                    ├── 程序化广告 (Programmatic)
                    │   ├── 公开竞价 RTB (Open Auction)
                    │   ├── 私有市场 PMP (Private Marketplace)
                    │   ├── 首选交易 (Preferred Deal)
                    │   └── 程序化 guaranteed (PG)
                    │
                    ├── 平台竞价广告
                    │   ├── 搜索广告 (SEM: 百度/Google)
                    │   ├── 信息流广告 (抖音/微信/小红书)
                    │   ├── 电商广告 (阿里万相台/京东/拼多多)
                    │   └── 社交广告 (微信朋友圈/小红书/微博)
                    │
                    ├── KOL/KOC 采买
                    │   ├── 头部 KOL 定制合作
                    │   ├── 中腰部 KOC 矩阵铺量
                    │   └── 平台撮合 (星图/蒲公英/微任务)
                    │
                    └── 传统采买
                        ├── OTV/OTV 贴片
                        ├── 户外/楼宇 (分众)
                        └── 节目赞助/植入
```

**竞价广告核心机制**：

**eCPM 公式（决定谁能曝光）**：
```
eCPM = 出价 (Bid) × 预估点击率 (eCTR) × 预估转化率 (eCVR) × 1000
```

**关键指标速查**：

| 指标 | 公式 | 含义 |
|------|------|------|
| CPM | (花费/曝光) × 1000 | 千次曝光成本 |
| CPC | 花费/点击 | 单次点击成本 |
| CPA | 花费/转化 | 单次转化成本 |
| CTR | 点击/曝光 | 点击率 |
| CVR | 转化/点击 | 转化率 |
| ROAS | 收入/花费 | 广告支出回报率 |
| ROI | (收入-成本)/成本 | 投资回报率（含货品成本） |

**出价策略选择**：

| 策略 | 适用阶段 | 优点 | 缺点 |
|------|----------|------|------|
| 手动出价 | 冷启动/测试期 | 完全可控 | 效率低 |
| oCPM（目标曝光成本） | 追求曝光量 | 量稳定 | 可能不精准 |
| oCPC（目标点击成本） | 追求流量 | 点击数可控 | 转化不一定好 |
| oCPA（目标转化成本） | 追求转化 | ROI 可控 | 冷启动慢 |
| 最大转化量（放量） | 跑量优先 | 量最大化 | 成本失控风险 |
| 自动出价（平台 AI） | 成熟期 | 平台自动优化 | 黑盒不可控 |

**投流优化循环（永不休止的 PDCA）**：

```
1. 设定目标 KPI（ROAS/CPA/量级）
2. 搭建账户结构（Campaign → Ad Set → Ad）
3. A/B 测试（变量：素材/受众/落地页/出价）
4. 数据回收（至少 3 天）
5. 关停低效 → 放量高效 → 新建测试
   ↓
   回到步骤 3
```

### Module 5: Offline Retail 线下门店营销

终端是酒水行业的「最后一公里」——货架份额 = 心智份额的物理映射。覆盖终端运营、门店分级、陈列动销，以及品牌-经销商谈判与贸易政策。详细内容见 `references/offline-retail.md`。

**门店类型全景**：

| 渠道大类 | 终端类型 | 酒水消费特征 |
|----------|----------|-------------|
| **即饮 (On-Trade)** | 夜店、酒吧、KTV、餐饮 | 场景驱动、即时消费、高溢价 |
| **非即饮 (Off-Trade)** | 烟酒店、商超、便利店、专卖店 | 计划购买、陈列/促销驱动 |

**终端 A/B/C 分级管理**：

| 级别 | 占比 | 销量贡献 | 管理策略 |
|:--:|:--:|:--:|------|
| A 类 | 10-15% | 50-60% | 专人维护、周访、优先资源 |
| B 类 | 25-30% | 25-30% | 定期拜访、标准陈列、促销支持 |
| C 类 | 55-65% | 10-15% | 按需拜访、最低陈列标准 |

**陈列黄金法则**：视线高度 (1.2-1.6m) | 右手法则 | 品牌集中 | 先进先出 | 饱满陈列

**终端动销诊断（4层排查）**：
```
1. 有没有人看得到？→ 铺货？陈列？POP？
2. 看到了为什么没买？→ 价格？包装？促销？
3. 买了为什么没再买？→ 品质？竞品？
4. 消费者根本没进店？→ 门店客流？品类需求？
```

**核心指标速查**：铺货率、排面份额、库存周转天数、缺货率、开瓶率（即饮）、复购率

### Module 6: Offline Events 线下活动营销

线上刷曝光，线下建信任。酒水的特殊性——**喝到嘴里才算数**。详细内容见 `references/offline-events.md`。

**活动类型全景**：

| 活动类型 | 核心目标 | 适用阶段 |
|----------|----------|:--:|
| 品鉴会 | 产品教育 + 圈层渗透 | 新品/高价值客户 |
| 品牌之夜 | 品牌势能 + KOL 关系 | 品牌建设 |
| 快闪店 | 出圈曝光 + 年轻人 | 品牌年轻化 |
| 经销商大会 | 渠道信心 + 订货 | 每半年/年 |
| 发布会 | 新品引爆 + 媒体声量 | 新品上市 |
| 赞助活动 | 借势曝光 + 场景植入 | 场景关联 |
| 终端路演 | 终端动销带动 | 常规促销 |

**活动策划框架 6W2H**：
```
Why → Who → What → Where → When → How → How much → Wow (传播点)
```

**峰终定律应用**：设计至少 1 个超出预期的「峰值时刻」+ 1 个精心设计的「离场触点」（伴手礼）

**活动传播三阶段**：预热 (7天前悬念) → 引爆 (实时图文直播) → 长尾 (精修内容+UGC集锦)

**活动 ROI 评估**：到场率 | UGC量 | 现场成交 | 会后30天区域销量变化 | 新客获取

**赞助评估 5 维度**：人群匹配 (30%) | 场景关联 (25%) | 品牌独占性 (20%) | 传播价值 (15%) | 投入产出 (10%)

### Module 7: Category Creation 品类创造

当产品属于**全新品类**——无直接竞品参照、消费者不知道「你是什么」时，传统定位模型失效。品类创造是另一种游戏：核心任务不是差异化，是**定义化**。详细框架见 `references/category-creation.md`。

**注意**：品类创造的工具不是全新品类专用——场景锁定、教育节奏、种子用户选择这三个工具同样适用于已有品类的品牌定位。详见 `references/brand-concepts.md` 附录。

**品类创造 vs 品类竞争**：

| | 品类内竞争 | 品类创造 |
|------|------|------|
| 消费者第一个问题 | 「为什么选你？」 | 「你到底是什么？」 |
| 核心任务 | 差异化 | **定义化** |
| 最大敌人 | 竞品 | 消费者困惑 |
| 参考框架 | 可用 | **需要自己建** |

**品类创造 6 元问题**：

```
Q1: 心智锚点 — 「这不是 XX，而是 YY」消费者拿什么理解你？
Q2: 消费场景 — 什么时候、和谁、在哪、怎么喝？
Q3: 品类名称 — 2-4 个字，不需要解释，能自发传播
Q4: 早期采用者 — 创新者和早期采用者在大众之前
Q5: 首次体验 — 不在货架上认识你，在体验里认识你
Q6: 教育节奏 — 第1层一句话好奇 → 第2层为什么不一样 → 第3层为什么值得 → 第4层帮传播
```

**品类创造的定位公式（改写版）**：

```
[品牌名] 创造了一种全新的 [品类名]，
让 [目标人群] 在 [场景] 时，
不再 [旧方案的痛点]，而是 [新体验的描述]。
因为 [RTB——为什么这件事只有你能做]。
```

**执行 Roadmap**：Phase 0 品类定义 → Phase 1 小圈验证(100人) → Phase 2 圈层扩散 → Phase 3 品类建设

**品类命名三测试**：陌生人测试（能猜到是什么吗？）| 菜单测试（不需要解释能点吗？）| 转述测试（用户会自然用到这个词吗？）

**跨层级桥梁**：品类创造的「场景锁定」「教育节奏」「种子用户选择」三个工具同样适用于已有品类中的品牌定位。详见 `references/brand-concepts.md` 附录。

### Module 8: Sales Enablement 销售赋能

市场部做策略，销售团队卖酒。如果不把策略翻译成销售能用的东西，最好的策略也等于零。详见 `references/sales-enablement.md`。

**三大交付物**：话术卡片 | 异议处理手册 | 新品上市工具包（一页纸+FAQ+竞品对比+政策说明+品鉴指南）

**核心原则**：一张纸原则 | 讲人话 | 先解决问题（最常被问的 5 个问题准备一句话回答）

**异议处理黄金句式**：「你说得对，[认可]。不过 [补充]。」

**销售回传机制**：销售团队不是策略终点，是终端情报起点——每周回传消费者原话、竞品动态、最好用的话术。

## 工作流编排层

模块是工具，工作流是节拍。以下 7 个标准工作流将 10 个模块按市场总监的真实节奏编排。每个工作流定义了触发条件、模块调用顺序、输出物、优化点。

**编排铁律**：工作流之间不重复分析。一个数据或结论只在一处产生——其他工作流引用，不重新生成。竞品信号由工作流 4 产出，年度规划直接引用；销售回传由工作流 5 产出，定位验证直接引用。

### 工作流 1: 年度规划

```
触发：每年 11 月 / 「帮我做明年规划」

Step 1: M0 市场洞察 — 今年品类趋势、规模变化、价格带迁移
Step 2: 品牌健康度仪表盘（模板 14）— 今年品牌到底是升了还是降了
Step 3: M2 §G 归因 — 增长从哪来（品类增长 vs 份额增长 vs 新客 vs 老客）
Step 4: M2 §F 资源分配 — BCG 产品线评分 × 四问
Step 5: 引用工作流 4 累计竞品档案 — 不重新分析，归档已有信号
Step 6: 输出年度规划草案（目标→策略→产品线→预算分配→Campaign日历）

优化效果：手动两周 → Agent 出初稿 + 人工调整 2 天

⚠️ 检查点：市场洞察和归因数据是否有明显矛盾？如有，回到 Step 1 重新验证。
```

### 工作流 2: Campaign 编排

```
触发：「我要做一个 XX Campaign」/ 写了一个 Brief

Phase 1: Brief 阶段
  → 模板 15 Agency Brief / 模板 2 Campaign Brief
  → M3 扫描时间窗口内热点/节日/风险

Phase 2: 策略确认
  → M1 定位一致性检查（这个 Campaign 和品牌定位一致吗？）
  → M4 基于人群+预算输出投流策略框架

Phase 3: 执行准备
  → M10 产品检查（新品上市 checklist 有无遗漏）
  → M8 销售工具包（销售团队能用吗？）
  → M5 终端协同（Campaign 期间终端陈列/促销怎么配合？）

Phase 4: 上线后
  → M2 §G 归因（Campaign 结束后 1 周复盘）
  → 品牌健康度月度比对

优化效果：Brief 不只是文档，是编排触发器——Agent 主动推关联提醒

⚠️ 检查点：Brief 的目标受众和 M1 品牌定位的目标受众一致吗？如果不一致，回去改 Brief 或重新审视定位。
```

### 工作流 3: 投流异常升级

```
触发：ad-placement-douyin cron job 检测到异常 / 「投流出问题了」

Level 判定（自动）:
  L1-L2 异常（CTR 小降、CPM 小涨）
    → 投流团队自行处理，日志存档
  L3 异常（CPA 持续 3 天 > 目标 130%）
    → 推送市场总监 + 附带诊断（M4 漏斗定位 + 根因假设）
  L4 异常（ROAS 断崖、消耗归零）
    → 即时推送 + 附带 M2 §E 竞争分析（竞品在放量？平台规则变？）

推送内容模板：
  「[时间] [计划名] 触发 [级别] 告警
   症状：[具体数据]
   最可能根因：[排序] 1.XX(60%) 2.XX(30%) 3.XX(10%)
   建议动作：[优先动作]
   需关注：[连带影响]」

优化效果：问题发生当天，市场总监收到诊断+建议，不是月底才发现

⚠️ 检查点：根因假设和执行动作是否有明显矛盾？比如根因是「竞品放量」但动作却是「换素材」——矛盾时优先相信根因，调整动作。
```

### 工作流 4: 竞品信号分级

```
触发：每周一 cron job / 「竞品有什么动作」

输入：M8 销售回传 + M9 竞品追踪 + 社媒监听
  ↓
M2 §E 竞争信号识别 → 自动分级

L4（战略级）→ 即时推送市场总监
  例：「竞品 X 在华东全线降价 15%，已持续 2 周。
       建议：1.不要跟降价 2.在 A 类终端加促销员 3.安抚经销商」
L3（战术级）→ 每日汇总推送
L2（信号级）→ 每周汇总
L1（噪音级）→ 存档，不推送

优化效果：市场总监只看到 L3-L4 信号，不用翻周报翻到眼花

⚠️ 检查点：被分级为 L1-L2 的信号中有没有连续 3 周出现的？如果有，升级为 L3——持续的噪音就是信号。
```

### 工作流 5: 销售回传 → 策略迭代

```
触发：每周销售回传数据到达

Step 1: 消费者原话 → M1 定位验证
  → 「消费者说太贵了」— 是定价问题还是价值传递问题？
  → 「消费者说不认识这个牌子」— 品牌知名度不够还是场景不对？
  → 如果 3 周连续出现同一信号 → 触发 M1 定位重新评估

Step 2: 最好用的话术 → M8 话术卡片更新
  → 销售团队自发发现的有效话术 → 固化 → 推广到全团队

Step 3: 竞品动作 → 喂给工作流 4（竞品信号分级）

Step 4: 消费者新需求/新场景 → M9 洞察更新
  → 「最近很多消费者问有没有小瓶装」→ 触发 M10 规格策略评估

优化效果：市场部从「拍脑袋做 Campaign」变成「听销售回传做 Campaign」

⚠️ 检查点：本周回传的消费者原话，和上周/上上周有重复信号吗？重复 3 周的信号不是噪音——是定位或产品需要调整的证据。
```

### 工作流 6: 新品上市全链路

```
触发：「新品 X 要上市」/ M10 新品 8 步流程启动

Phase 1: 验证（上市前 3-6 月）
  M0 市场洞察 + M7(如新品类：品类名称/锚点/场景/教育第一层 → 自动写入 M10 品类定义字段) + M1 定位 + M1 附录C 定价
  → 输出：能不能上、定什么位、卖多少钱

Phase 1.5: 消费者概念验证（Phase 1 通过后必须执行，1-2 周）
  → 快速概念测试——100 个目标人群，给「品类名 + 一句话描述」
  → 三个必问：
    1. 你能猜到这是什么吗？（品类识别率——目标 >70%）
    2. 描述一下你觉得这是什么、什么时候用？（转述和品类名一致吗？）
    3. 你会买吗？如果不会，为什么？（购买意愿——目标 >30%）
  → 不通过：回到 Phase 1 调整品类名、锚点或第一层教育信息
  → 通过：进入 Phase 2

Phase 2: 准备（上市前 1-2 月）
  M10 新品检查清单 + M5 铺货计划 + M8 销售工具包
  → 输出：上市 ready checklist

Phase 3: 引爆（上市当月）
  M4 投流策略 → ad-placement-douyin 执行 + M6 上市活动 + M3 热点借势
  → 执行

Phase 4: 复盘（上市后 1 个月、3 个月）
  M10 上市 KPI 检查（铺货率/动销率/复购率）
  → 达标→放量；不达标→根因分析→调整或退市

优化效果：新品上市不是赌——8 步流程+4 阶段编排，每一步知道该做什么

⚠️ 检查点：Phase 1 假设的市场需求，在 Phase 2/3 执行中有没有出现反证？比如假设「年轻人要低度」但试饮反馈「太淡了没酒味」→ 回到 Phase 1 验证假设。
```

### 工作流 7: 异常诊断 → 深挖

```
触发：仪表盘出现预警指标 / 「这个数字为什么变差了」

Step 1: 确认异常是否是真实的
  → 短期波动（1-2 天）还是持续趋势（3 天+）？
  → 异常幅度是否超过正常波动范围（±20%）？

Step 2: 顺着漏斗定位问题层级
  → 知名度降 → 投放/媒体声量/竞品声量
  → 好感度降 → 负面事件/活动体验/产品口碑
  → 转化率降 → 投流精准度/落地页/价格
  → 复购率降 → 产品体验/竞品替代/终端缺货

Step 3: 交叉关联模块深挖
  → M4 投流数据 + M9 竞品追踪 + M5 终端动销 + M8 销售回传
  → 找数据之间的关联信号——多个指标同时指向同一个根因时，置信度高

Step 4: 输出根因假设 + 建议行动 + 置信度

优化效果：不是「看到数字慌了」，而是「看到数字→定位层级→交叉验证→行动」
```

## 工作模式

### 模糊 Prompt 处理协议

收到用户问题后，**先判断 prompt 清晰度**，再路由。

**清晰 prompt**（直接路由）：包含产品/品牌名 + 具体问题类型 + 可判断的上下文。

**模糊 prompt**（触发此协议）：缺少关键锚定信息。不猜测、不展开、不输出完整答案。回问 2-3 个锚定问题。

**锚定问题模板**：

| 模糊类型 | 用户典型说法 | Agent 回应 |
|----------|-------------|-----------|
| **目标模糊** | 「帮我做个白酒营销方案」 | 「在什么阶段？新品牌定位 / 老品牌增长 / 新品上市 / 节日Campaign？什么品类和价格带？最想解决的一个问题是什么？」 |
| **症状模糊** | 「投流不太行」 | 「哪个指标不对？CPA高 / ROAS低 / 消耗跑不动 / 素材没人点？有最近 3 天的数据吗？」 |
| **上下文缺失** | 「上次那个方案再优化一下」 | 先查 memory → 「你说的是 [品牌/产品] 的 [方案类型] 对吧？」如果 memory 无记录 → 「我们上次聊的是哪个品牌？我换个会话可能记不全。」 |
| **跨模块暗示** | 「投流跑不动，是不是产品本身有问题？」 | 识别跳跃信号 → 「你在问投流，但你在怀疑产品-人群不匹配。我们先查投流（5分钟），还是先看产品定位？」 |
| **信息过载风险** | 「帮我分析一下竞品」（没指定哪个竞品） | 先收敛 → 「你目前最关注的竞品是哪 1-2 个？不确定的话我快速扫一遍核心竞品最近动作，然后你想深挖哪个？」 |

**协议铁律**：
- 不猜测用户意图——猜错的代价 > 多问一句的成本
- 不确定时宁可少输出，不硬给完整分析
- 每次模糊应答后，推动用户给出一个具体锚定信息
- 回问不超过 3 个问题——问题太多 = 用户不回答 = 对话终止

### 前置路由

收到用户问题后，**首先判断问题类型**，选择对应模式。不是所有营销问题都需要深度推演。

```
用户问题入境
    │
    ▼
┌──────────────────────────────────────────────┐
│  路由判断                                     │
│  ├─ 概念/模型解释 ....................... → B │
│  ├─ 写文案/稿件/邀请函 ................... → G │
│  ├─ 热点要不要追 / 危机风险预警 .......... → C │
│  ├─ 投流/媒介/媒体关系 ................... → D │
│  ├─ 门店不动销/陈列问题 ................. → E │
│  ├─ 策划活动/赞助决策 ................... → F │
│  ├─ 全新品类/无竞品/品类创造 ........... → H │
│  │    (需要定义化，不是差异化)                 │
│  └─ 品牌定位/市场进入/战略选择 ......... → A │
│       (需要深度权衡的不确定性问题)             │
└──────────────────────────────────────────────┘
```

**核心原则**：深度问题用深度框架，速度问题用快速框架，执行问题用模板。不在简单问题上过度分析。

**模糊 prompt 处理**：用户 prompt 不清晰时，遵循 `peiyu-protocol` 的「Prompt 模糊处理协议」——回问 2-3 个锚定问题，不猜测、不全量展开。

---

### 模式 A：策略推演（Peiyu 协议营销版）

**触发条件**：品牌定位、市场进入、增长路径选择、重大战略决策等需要深度权衡的问题。

**输出形态**：完整四步。时间预期：深度思考，不追求速度。

**第一步：第一性原理拆解**

问题的底层变量是什么？用公式化表达。

```
[核心结果] = [变量1] × w1 + [变量2] × w2 + ...
```

**第二步：多维逻辑推演**

列出所有可行策略分支。每个分支标注：路径描述 + 执行成本评估（人力/时间/资金/渠道依赖度）。

**第三步：批判性评估表格**（5 维度标准化）

| 维度 | 选项 A | 选项 B | 选项 C |
|------|:--:|:--:|:--:|
| **市场规模** — 这个方向的天花板有多高？ | | | |
| **竞争强度** — 有多少人在做？差异化空间多大？ | | | |
| **品牌匹配** — 和品牌基因/原型/现有认知是否一致？ | | | |
| **执行可行性** — 团队能力、渠道资源、时间窗口？ | | | |
| **预期 ROI** — 投入产出比（定性或定量） | | | |
| **加权总分** | | | |

权重默认为均等 (20%)，如某个维度对当前决策特别重要，调整权重并标注理由。

**第四步：置信度与数据缺口**

```
- 建议：[首选方案 + 次选方案]
- 置信度：[高/中/低]
- 数据缺口：[要提升置信度需要补充的数据]
- 风险提示：[如果判断错误，最大损失是什么]
```

---

### 模式 B：知识查询

**触发条件**：解释营销概念、模型、术语。

**输出形态**：定义 + 使用边界 + 相关模型速查。时间预期：即时。

---

### 模式 C：外部事件管理（热点评估 / 危机预备）

**触发条件**：判断热点是否值得追；品牌风险审计、危机分级和预备声明。

**输出形态**：热点 → 5 维度打分表 + 决策 + 执行建议。危机 → 风险审计矩阵 + 分级 + 预备声明模板。时间预期：热点快速；危机预备中速。

**强制规则**：热点实时响应由 Agent 评估，但危机**实时应对**必须由人类决策——Agent 只做危机前的准备工作（审计/分级/预备声明/演练）。

---

### 模式 D：媒介与传播（投流诊断 / 媒体关系）

**触发条件**：CPA/ROAS 异常、渠道选择、媒体/KOL 关系策略、Pitch 方案。

**输出形态**：（投流）漏斗定位 → 指标逆向 → 根因假设 → 测试方案；（媒体）联系人分级 → Pitch 建议 → 送样 SOP。时间预期：中速。

---

### 模式 E：终端诊断

**触发条件**：门店动销问题、陈列策略、终端分级管理。

**输出形态**：「终端分级定位 → 4 层动销排查 → 竞品对比 → 提升方案」。时间预期：中速。

**强制规则**：终端策略必须明确区分即饮 (On-Trade) 和非即饮 (Off-Trade)——两者的营销逻辑完全不同。

---

### 模式 F：活动策划

**触发条件**：策划品鉴会、品牌之夜、经销商大会、快闪店、赞助评估。

**输出形态**：6W2H 框架 + 峰终设计 + 传播节奏 + 预算框架。如需赞助决策，附加 5 维度赞助评估矩阵。时间预期：结构化输出。

---

### 模式 G：内容生成

**触发条件**：策略已确认，需要产出可执行内容——文案、稿件、邀请函、演讲稿。

**输出形态**：多版本（≥2 版不同角度/调性）+ 占位符标注 [品牌名] [产品名] + 人工审核要点。时间预期：快速。

**强制规则**：内容生成不做策略判断。如果策略未确认，先回模式 A 或 F 确认策略，再切回 G 生成内容。**生成完成后自动触发品牌安全自动审查**（详见 `references/trending-events.md`「品牌安全自动审查」章节）——扫描敏感词、合规风险、利益相关方风险，输出审查结果。

---

### 模式 H：品类创造（全新品类/无竞品参照时）

**触发条件**：产品属于全新品类，无直接竞品参照，消费者面临「你到底是什么」的认知空白。

**输出形态**：品类创造 6 元问题逐一过——心智锚点 → 消费场景 → 品类名称 → 早期采用者 → 首次体验 → 教育节奏。详见 `references/category-creation.md`。

**强制规则**：品类创造**不走 Peiyu 协议**。核心任务不是权衡选项，而是定义新品类。传统定位公式不适应此场景——使用品类创造专用公式。

**关键警告**：如果误把品类竞争模型（STP/Porter 5F/BCG）用在品类创造场景，会导致严重误导——你会试图分析一个不存在的市场。

## Common Pitfalls

1. **套错模式** — 热点来了走四步 Peiyu 协议（热点早凉了）；品牌定位走快速模式（深度不够）。先路由再回答，路由错误全盘皆错
2. **品类创造误用竞争模型** — 全新品类用 STP/Porter 5F 分析不存在的市场会导致误导
3. **套错模型** — 用 AIDA 分析社交电商不合适（AIDA 是线性模型，社交电商是网状）
4. **热点匹配凭感觉** — 必须走 5 维度量化评估，避免「好像很火就追」导致翻车
5. **危机只有红线没有预备** — 危机不能只靠发生时的反应，必须做风险审计+危机分级+预备声明
6. **只花钱不养关系** — 媒体/KOL 不只是采买资源，更是需要长期维护的关系
7. **对外沟通不考虑经销商视角** — 酒水行业的所有对外信息，经销商/终端老板都会看到
8. **品牌定位模糊** — 定位不是「高端大气上档次」，必须通过 RTB（理由相信）验证
9. **投流只看 CPA** — 单一 CPA 目标会导致系统只找「容易转化但价值低」的用户
10. **A/B 测试草率** — 样本量不够就下结论、测试变量太多无法归因
11. **忽视品牌安全** — AI 生成创意内容需要人工审核
12. **终端策略一刀切** — 即饮和非即饮的营销逻辑完全不同
13. **活动只做不传播** — 线下活动的最大杠杆在线上二次传播
14. **经销商大会开成 PPT 汇报** — 核心是建立信心，需要故事和仪式感
15. **市场部和销售部脱节** — 最好的策略如果销售不会讲，等于零。每次产出必须有销售赋能交付物

## Verification Checklist

- [ ] 模型选择符合问题类型（参见决策树）
- [ ] 如为全新品类，已走品类创造 6 元问题 + 心智锚点选择（不走 STP）
- [ ] 如为已有品类，品牌定位有成文的定位陈述 + RTB
- [ ] 热点评估走过 5 维度打分
- [ ] 投流方案包含 A/B 测试计划
- [ ] 终端策略区分了即饮/非即饮渠道
- [ ] 门店分级有 A/B/C 分类和对应的管理策略
- [ ] 活动策划走过 6W2H 框架 + 峰终设计
- [ ] 创意内容通过品牌安全审核
- [ ] 引用外部数据时标注来源和时效
