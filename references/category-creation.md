# Category Creation — 品类创造框架

> 加载时机：新品属于全新品类、无直接竞品参照、需要定义而非差异化时。
> 核心洞见：品类创造和品类竞争是两种完全不同的游戏。前者需要「定义化」，后者需要「差异化」。

---

## 1. 品类创造 vs 品类竞争

| | 品类内竞争 | 品类创造 |
|------|------|------|
| 消费者第一个问题 | 「为什么选你而不是竞品？」 | **「你到底是什么？」** |
| 核心任务 | 差异化 | **定义化**——给新事物名字和坐标系 |
| 竞争对手 | 同品类其他品牌 | **消费者的旧习惯和旧认知** |
| 增长逻辑 | 抢份额 | **做蛋糕** |
| 参考框架 | 可用——「比 XX 更好」 | 不存在——需要**自己建** |
| 最大敌人 | 竞品 | **消费者困惑**：「这什么？什么时候喝？怎么喝？」 |
| 适用模型 | STP / Porter 5F / SWOT / BCG | **本框架的 6 元问题** |
| 仍可用的模型 | — | JTBD / 品牌原型 / PESTEL / 峰终定律 / Kapferer 棱镜 |

---

## 2. 品类创造的 6 个元问题

```
Q1: 心智锚点 — 消费者拿什么来理解你？
  ↓
Q2: 消费场景 — 什么时候、和谁、怎么喝？
  ↓
Q3: 品类名称 — 你管自己叫什么？
  ↓
Q4: 早期采用者 — 谁最可能第一个尝试？
  ↓
Q5: 首次体验 — 消费者怎么第一次喝到你？
  ↓
Q6: 教育节奏 — 先教什么，后教什么？
```

## 3. Q1: 心智锚点

人类无法理解全新事物。所有新品类必须**锚定在已知品类上**，然后说「我像 X，但不一样在 Y」。

### 锚点公式

```
「这不是 [已知品类]，而是 [差异化描述]」
```

### 经典案例

| 新品类 | 锚点选择 | 锚定句式 |
|--------|----------|----------|
| 特斯拉 Roadster | 跑车 | 「这不是电动车，是不加油的跑车」 |
| 红牛（美国市场） | 汽水 | 「这不是汽水，是功能饮料」 |
| 气泡水 | 矿泉水 | 「这不是矿泉水，是有味道没糖的水」 |
| 燕麦奶 | 牛奶 | 「这不是牛奶，是植物做的奶」 |
| 酱香拿铁 | 咖啡 | 「这不是普通拿铁，是加了茅台的拿铁」 |

### 锚点选择矩阵

对 X（纯植物草本中度鸡尾酒）的锚点评估：

| 锚定选项 | 消费者第一反应 | 优势 | 风险 |
|----------|---------------|------|------|
| 「这是一种鸡尾酒」 | 「预调酒那种？RTD？」 | 场景明确（派对/酒吧） | 被 RTD 认知绑架，差异化被抹平 |
| 「这是一种草本酒」 | 「像劲酒？药味重吗？」 | 差异化强 | 保健酒/药酒刻板印象极难洗 |
| 「这是一种植物基酒饮」 | 「植物基？类似燕麦奶的逻辑？」 | 借势植物基趋势，健康联想 | 需要教育「植物基酒饮」这个子品类 |
| 「这是一种新的微醺方式」 | 「微醺我知道，但怎么不一样？」 | 最开放的解释空间 | 锚点最模糊，教育成本最高 |

**选择锚点的本质**：不是选哪个最准确，是选**你想让谁做第一批用户**。锚定鸡尾酒 → 已有饮酒习惯的人。锚定植物基 → 健康意识强的人。

## 4. Q2: 消费场景

已有品类的场景是文化沉淀出来的（白酒=宴请、啤酒=聚会）。新品类没有文化沉淀，必须**主动定义和传播场景**。

### 场景定义矩阵

| 场景 | 情绪 | 适合品类特质 |
|------|------|-------------|
| 独处放松 | 安静、自我奖赏 | 不烈不甜、有仪式感 |
| 朋友小聚 | 轻松、社交 | 好喝不挑人、有分享性 |
| 户外/自然 | 自由、探索 | 便携、和自然联想 |
| 餐前开胃 | 精致、期待 | 清爽、唤醒味蕾 |
| 睡前助眠 | 舒缓、放松 | 低度、天然成分 |
| 创意调饮 | 好玩、个性化 | 适合做基酒调配 |

### 场景选择的战略原则

**先打透一个场景，再扩展。** 新品类需要极其清晰的「使用说明书」。同时打五个场景 = 消费者不知道怎么用 = 不买。

### 场景的「使用说明书」模板

```
[时间] + [和谁] + [在哪] + [怎么喝] + [什么感觉]

示例：
「周五晚上，一个人在家，
 放一首歌，倒半杯 [X] 加冰，
 草本香气慢慢散开，不烈不甜——这就是你的一周犒赏。」
```

## 5. Q3: 品类名称

品类创造最被低估的战略决策。规则：**2-4 个字，消费者能自发说出来，不需要解释。**

### 品类命名的三条铁律

1. **可自传播**：消费者在餐桌上能自然说出「来一瓶 XX」——不需要解释这是什么
2. **不撞名**：不能和现有品类/品牌产生混淆
3. **有联想**：名字本身暗示品类特征（如「气泡水」暗示有气泡）

### 命名策略

| 策略 | 示例 | 适用 |
|------|------|------|
| 借势型 | 植物奶（借「奶」的认知） | 有强锚定品类时 |
| 特征型 | 气泡水（直接描述特征） | 核心特征单一且独特 |
| 场景型 | 微醺酒（场景即品类） | 场景是核心差异化 |
| 原料型 | 梅酒（原料=品类） | 原料有强认知 |
| 创新词 | 拿铁、卡布奇诺（音译创新） | 完全无锚点的品类 |

### 命名测试

一个好品类名必须通过三个测试：
1. **陌生人测试**：跟一个不知道你品牌的人说品类名，TA 能猜到大概是什么吗？
2. **菜单测试**：放在餐厅酒单上，消费者不靠任何解释能点吗？
3. **转述测试**：用户喝完以后跟朋友描述时，会自然用到这个词吗？

## 6. Q4: 早期采用者

跨越鸿沟理论：新品类不能一开始就瞄准主流人群。必须先找到**创新者 + 早期采用者**。

```
创新者 (2.5%) → 早期采用者 (13.5%) → 早期大众 (34%) → 晚期大众 (34%) → 落后者 (16%)
     ↑                ↑
  首要目标          次要目标          ← 鸿沟之前，不要碰大众
```

### 早期采用者筛选标准

| 标准 | 为什么重要 |
|------|-----------|
| **可达性** | 能否通过特定渠道/社群接触到？ |
| **尝试意愿** | 对新鲜事物的开放度？ |
| **传播力** | 尝试后会主动分享吗？有圈层影响力吗？ |
| **容忍度** | 对产品不完美的容忍度？（早期产品一定有缺陷，大众用户零容忍） |
| **需求强度** | 你的产品是否解决了 TA 的某个真实痛点？ |

### 早期用户画像模板

```
- 人口特征: [年龄/性别/城市/收入]
- 生活方式: [兴趣圈子/消费习惯/信息获取渠道]
- 现有替代方案: [TA目前用什么解决这个问题]
- 痛点: [现有方案让TA哪里不满意]
- 传播触点: [TA在哪个平台/社群获取新品信息]
- 一句话描述: [让团队所有人都能一眼认出这个用户]
```

## 7. Q5: 首次体验

消费者不是在货架上认识你的，是在一个**体验**里认识你的。货架认识 = 和 500 个 SKU 竞争注意力。体验认识 = 你把场景和情绪一起打包给了 TA。

### 首次体验的四种路径

| 路径 | 适合品类 | 关键设计 |
|------|----------|----------|
| **活动场景** | 社交属性强的品类 | 音乐节/市集/露营——在年轻人聚集地做试饮 |
| **餐饮渠道** | 和用餐场景关联的品类 | 找 5 家调性匹配的餐厅，列入特别推荐 |
| **KOL/KOC 种草** | 适合内容呈现的品类 | 不是硬广，是「发现了一种没喝过的酒」 |
| **朋友推荐** | 社交货币强的品类 | 设计分享机制（买一送一、UGC 激励） |

### 首次体验设计的峰终定律

```
体验全程
  ├── 拿到手的第一刻（包装、重量、温度）← 第一印象
  ├── ★ 峰：入口/打开那一瞬间 ← 设计惊喜
  ├── 品尝过程
  └── ★ 终：喝完的感觉 + 离开时带走什么 ← 最后记忆
```

### 体验设计检查清单

- [ ] 包装能否引发「这是什么」的好奇？
- [ ] 入口的第一秒和预期一致还是超出预期？
- [ ] 体验过程中有没有值得拍照/分享的瞬间？
- [ ] 结束后有没有一句话可以让用户转述给朋友？
- [ ] 有没有给用户一个「再来一次」的理由？

## 8. Q6: 品类教育节奏

品类教育的最大错误：一次性把所有信息塞给消费者。

### 四层递进模型

```
第 1 层：认知（第一次接触）
  只说一件事 → 让人好奇
  「这是一种新的喝法。草本+微醺，好喝不甜。」
  ↓
第 2 层：理解（引发了兴趣后）
  回答「为什么不一样」
  「不用香精，纯植物提取。不是预调酒，不是保健酒。」
  ↓
第 3 层：信服（准备购买时）
  回答「为什么值得」
  品牌故事、原料产地、工艺细节、社会证明
  ↓
第 4 层：传播（成为用户后）
  让他去帮你教育别人
  「你喝的这是什么？」← 这个答案，用户必须能一句话转述
```

### 每层对应的媒介

| 层 | 媒介 | 信息量 | 目标 |
|:--:|------|:--:|------|
| 1 认知 | 短视频/社媒帖子/朋友推荐 | 一句话 | 好奇→搜索 |
| 2 理解 | 详情页/KOL 测评/公众号 | 300-500 字 | 理解→考虑 |
| 3 信服 | 品鉴会/试用装/口碑 | 完整体验 | 考虑→购买 |
| 4 传播 | UGC/社交分享 | 用户原话 | 购买→推荐 |

### 教育节奏的常见错误

1. **第 1 层就讲故事**：消费者还不知道你是什么，品牌故事是第 3 层的事
2. **跳过第 2 层直接卖**：消费者不知道「为什么不一样」时不会买
3. **第 4 层没有准备**：用户想帮你传播，但不知道怎么一句话说清楚——这是你的责任，不是用户的

## 9. 品类创造的执行 Roadmap

```
Phase 0: 品类定义（1-2 月）
  ├── Q1 选定心智锚点
  ├── Q2 确定核心场景
  ├── Q3 锁定品类名称
  └── Q4 锁定早期用户画像

Phase 1: 小圈验证（3-6 月）
  ├── Q5 设计首次体验
  ├── 找到 100 个早期用户
  ├── 收集反馈（不是好评，是「你什么时候喝、怎么喝的」的真实数据）
  └── 迭代产品/包装/信息

Phase 2: 圈层扩散（6-12 月）
  ├── Q6 执行教育节奏第 1-2 层
  ├── 早期用户 → 用户的社交圈
  └── 进入 2-3 个核心渠道（不是铺货，是精选）

Phase 3: 品类建设（12-24 月）
  ├── Q6 执行教育节奏第 3-4 层
  ├── 品类名在目标人群中自然流通
  └── 开始有「XX 品类」的媒体报道和行业讨论
```

## 10. 品类创造下的品牌定位公式

当品类不存在时，定位公式需要改写：

**旧公式（品类内）**：
```
对于 [目标人群]，[品牌名] 是 [品类] 中
能够提供 [差异化利益] 的品牌，因为 [RTB]。
```

**新公式（品类创造）**：
```
[品牌名] 创造了一种全新的 [品类名]，
让 [目标人群] 在 [场景] 时，
不再 [旧方案的痛点]，而是 [新体验的描述]。
因为 [RTB——为什么这件事只有你能做]。
```

### 示例

```
旧公式思维（错）：
「X 是草本酒中能够提供纯植物微醺体验的品牌」
→ 问题：草本酒这个品类还不存在，消费者不知道你在跟谁比

新公式思维：
「X 创造了一种全新的植物微醺，
 让你在独处放松时，
 不再被酒精的负担感和甜腻感困扰，
 而是享受草本自然的轻快愉悦。
 因为只有我们用 7 种植物冷萃提取，不加一滴香精。」
```
