# Sales Enablement — 销售赋能

> 加载时机：市场部产出策略/内容后，需要交付给销售团队和经销商执行时。桥接「策略」和「执行」的最后一环。
> 核心理念：最好的策略如果不能被销售团队一句话讲清楚，等于不存在。

---

## 1. 销售赋能的设计原则

市场部做 Campaign、做内容、做投放——但最终卖酒的人是销售团队和终端业务代表。销售赋能就是把市场部的产出翻译成**销售团队能在终端直接用的东西**。

**三条铁律**：
1. **一张纸原则**：销售团队最常用的东西不能超过一页纸。信息密度 ≠ 信息量
2. **讲人话**：不是把品牌手册复印缩小，是把核心信息翻译成消费者能听懂的话
3. **先解决问题**：销售最常被问到的 5 个问题，每个准备一句话回答

---

## 2. 销售话术卡片

终端业务代表口袋里或手机里随时能掏出来的东西。

### 卡片设计原则

- 正面：一句话推荐 + 三个卖点（消费者语言）
- 背面：五个常见异议的回答

### 模板

```
┌─────────────────────────────────┐
│  [品牌名] — 一句话推荐           │
│                                 │
│  「[品牌名]，[品类]里 [一句话差异]│
│   的那个。尝一下？」              │
│                                 │
│  三个卖点（消费者语言）：         │
│  1. [不是工艺参数，是好处]        │
│  2. [不是原料名字，是体验]        │
│  3. [不是获奖信息，是感觉]        │
│                                 │
│  适合推荐给：[人群特征]           │
│  最佳推荐时机：[场景]            │
├─────────────────────────────────┤
│  常见问题（背面）                │
│                                 │
│  Q: 太贵了                      │
│  A: [不是「不贵」，是「为什么值」]│
│                                 │
│  Q: 没听说过                     │
│  A: [不是「我们很有名」，         │
│      是「为什么值得试」]          │
│                                 │
│  Q: XX 品牌更好                  │
│  A: [不是贬低竞品，是说差异]      │
└─────────────────────────────────┘
```

### 酒水行业示例

```
正面：
「江小白，白酒里最懂年轻人的那个。尝一下？」

三个卖点：
1. 入口不辣——不像传统白酒那样呛
2. 可以加冰喝——不用非得小口抿
3. 瓶子好看——拍照发朋友圈不丢人

适合推荐给：25-35 岁、朋友聚会场景
最佳推荐时机：客人点饮料时说「要不要试试这个？」

背面：
Q: 太贵了
A: 比同价位白酒好入口，第一次喝白酒的人都能接受

Q: 没听说过
A: 年轻人现在都喝这个——小红书上一搜就知道了

Q: 江小白不如 XX 白酒好
A: XX 是传统白酒，这个是新喝法——不一样的体验
```

## 3. 异议处理手册

终端最常见的 6 类异议 + 标准回应框架。

| 异议类型 | 消费者原话 | 回应原则 | 示例 |
|----------|-----------|----------|------|
| **价格** | 「太贵了」 | 不否认价格，说价值 | 「它比普通的多了一个[差异点]，你试一下就知道了」 |
| **陌生** | 「没听说过」 | 不辩解名气，给理由试 | 「它是[人群/场景]现在最火的，你可以先试一口」 |
| **竞品** | 「XX 品牌更好」 | 不贬低竞品，说差异化 | 「XX 是[方向A]，我们是[方向B]——看你要哪种感觉」 |
| **习惯** | 「我平时不喝这个」 | 不否定习惯，给新选项 | 「试试看不亏——很多人第一次喝都是这个反应，然后就回不去了」 |
| **场合** | 「这个场合不合适」 | 重新定义场合 | 「其实[新用法]也可以——不一定非得[旧用法]」 |
| **健康** | 「喝酒不健康」 | 不反驳，轻度引导 | 「少喝点、喝好点——这个[天然/低度/无添加]」 |

### 异议处理的黄金句式

```
「你说得对，[认可情绪/事实]。不过 [补充信息]。」
```

不是「你不懂」，是「你说得对，不过还有一个角度」。

## 4. 新品上市销售工具包

市场部在新品上市前必须交付给销售团队的一整套东西。

| 工具 | 内容 | 篇幅 | 给谁 |
|------|------|:--:|------|
| **一页纸** | 一句话定位 + 三个卖点 + 目标消费者 + 推荐场景 | 1 页 | 所有销售人员 |
| **话术卡片** | 推荐语 + 异议回答 | 正反一页 | 终端业务代表 |
| **FAQ** | 消费者/经销商常问 20 问 + 标准回答 | 2-3 页 | 销售经理/城市经理 |
| **竞品对比表** | 本品 vs 3 个主要竞品在价格/口感/场景/卖点上的对比 | 1 页 | 经销商谈判 |
| **政策说明** | 返利政策/陈列支持/促销计划/新品铺货激励 | 1 页 | 经销商 |
| **品鉴指南** | 怎么喝（温度/杯具/配餐）、品鉴要点、引导语 | 1 页 | 终端促销员/BA |

### 工具包检查清单

- [ ] 销售人员能在 30 秒内讲清楚这个新品是什么、卖给谁、为什么好卖
- [ ] 经销商能在 3 分钟内算清楚「做这个新品一年能多赚多少钱」
- [ ] 终端 BA 能在消费者犹豫时用 1 句话打消顾虑
- [ ] 所有材料中消费者的描述是一致的（不出现销售人员说 A、广告说 B）

## 5. 销售团队培训设计

| 培训环节 | 内容 | 方式 | 时长 |
|------|------|------|:--:|
| 品牌故事 | 不是念品牌手册——让销售团队**喝到、感受到** | 品鉴+故事讲述 | 30min |
| 产品知识 | 核心卖点 + 和竞品的真实差异 | 对比品鉴+问答 | 30min |
| 话术演练 | 角色扮演——一个人演消费者，一个人用话术接 | 实战演练 | 45min |
| 异议处理 | 6 类异议逐一过关 | 模拟+复盘 | 30min |
| 政策通关 | 返利怎么算、促销怎么用——必须算对 | 算账练习 | 20min |

**培训的底线**：培训结束前，每个人必须做到——不参考任何材料，用自己话说出：这个产品是什么、卖给谁、为什么好卖。做不到的人不通过。

## 6. 销售回传机制

销售赋能不是单向的——销售团队是品牌在终端的眼睛和耳朵。

| 回传信息 | 频率 | 格式 | 用途 |
|------|:--:|------|------|
| 本周最常被问到的 3 个问题 | 每周 | 一句话 | 更新话术/FAQ |
| 竞品动态 | 每周 | 竞品降价/新品/促销 | 竞争情报 |
| 终端反馈 | 每周 | 消费者/店主原话 | 迭代产品/包装/信息 |
| 卖得最好的话术 | 每月 | 原句 | 推广到全团队 |

**一句话铁律**：销售每周反馈回来的消费者原话，是市场部最宝贵的策略输入——比任何调研报告都真实。
