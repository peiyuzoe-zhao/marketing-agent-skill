# Trending Events — 热点营销事件机制

> 加载时机：判断热点是否值得追、网络热点与品牌策略匹配、热点营销执行方案设计。

---

## 1. 热点的本质

热点 = 公共注意力在特定时间窗口内的高度集中。
营销借势 = 将品牌的商业信息附着在公共注意力流上，以远低于常规投放的成本获得关注。

**底层公式**：
```
热点营销 ROI = (热点自然流量 × 品牌关联度 × 内容质量) / (反应速度成本 + 翻车风险折损)
```

---

## 2. 热点生命周期与介入策略

```
 热度
  ↑
  │        ╱‾‾‾‾‾‾‾╲
  │      ╱            ╲
  │    ╱                ╲
  │  ╱                    ╲________
  │╱                                ╲___
──┼─────────────────────────────────────→ 时间
  潜伏   爆发   峰值    衰退    长尾   消亡
  (0-6h)(6-24h)(24-48h)(48-72h)(3-7d)(7d+)
```

| 阶段 | 特征 | 策略 | 风险 |
|------|------|------|------|
| **潜伏期** | 话题在小圈子发酵，尚未出圈 | 最早介入——如果你在圈内且品牌高度匹配 | 可能判断失误，话题根本没火 |
| **爆发期** | 快速扩散，话题热度曲线陡升 | **最佳窗口**——快速评估 + 快速产出 + 快速发布 | 反应稍慢就错过峰值 |
| **峰值期** | 全民讨论，所有品牌都在追 | 仍可参与，但竞争激烈，需要更独特的切入角度 | 内容淹没在噪音中 |
| **衰退期** | 热度下降，公众注意力转移 | 不建议追，除非你能给出「总结性/升华性」内容 | 给人「反射弧太长」的负面印象 |
| **长尾期** | 偶有提及，但已无脉冲式流量 | 不做热点营销，可做案例复盘 | 追了像「考古」 |

## 3. 热点-品牌匹配度评估矩阵（核心工具）

对任何热点，从 5 个维度打分（1-5），加权计算：

| 维度 | 评估内容 | 权重 | 打分标准 |
|------|----------|:--:|------|
| **相关性** | 话题和品牌/品类/产品有无自然连接？ | 30% | 5=天然相关, 3=可以牵线, 1=毫无关系 |
| **调性匹配** | 热点情绪基调是否匹配品牌人设？ | 25% | 5=完美契合, 3=中性, 1=严重冲突 |
| **风险度** | 反转概率？敏感度？可能被解读为「消费XX」？ | 25% | 5=零风险, 3=有风险但可控, 1=高危 |
| **时效窗口** | 还剩多少时间有效参与？ | 10% | 5=刚爆发, 3=峰值中, 1=已衰退 |
| **传播势能** | 扩散速度和广度？出圈层级？ | 10% | 5=全民级, 3=圈层级, 1=小圈子 |

**综合分 = Σ(各维度分 × 权重)**

**决策阈值**：
- **≥ 4.0** → 全力追：全资源投入，最高优先级
- **3.0-3.9** → 选择性参与：轻量级内容，不投入大资源
- **2.0-2.9** → 观望：不做内容，但关注演变，可能升级
- **< 2.0** → 放弃：不参与

**放弃≠永远不参与**：热点可能演变出新的角度，原先不匹配的可能变为匹配。持续监控。

## 4. 热点「红线」——绝对不碰

| 红线类型 | 原因 | 案例教训 |
|----------|------|----------|
| **灾难/事故** | 消费灾难是最低级错误 | 某车企用地震做营销翻车 |
| **政治敏感** | 品牌站队=自杀 | — |
| **社会伤痛事件** | 缺乏共情力的品牌被永久钉上耻辱柱 | 某品牌在公祭日发促销 |
| **竞品主导话题** | 追 = 免费帮竞品做传播 | — |
| **品牌核心价值观冲突** | 短期流量换长期品牌损害 | 高端品牌追低俗梗 |
| **版权/肖像权不明** | 法律风险 | 随意使用网络梗图被诉 |
| **尚未定论的争议事件** | 容易反转打脸 | — |

## 5. 热点监控体系

### 5.1 监控源

```
实时监控层（每小时扫）：
├── 微博热搜榜 (https://weibo.com/hot)
├── 抖音热点榜
├── 百度热搜
└── 知乎热榜

行业相关层（每日扫）：
├── 行业媒体/公众号
├── 竞品社媒动态
├── 行业 KOL 内容
└── 小红书行业话题

趋势预警层（每周分析）：
├── 百度指数趋势
├── 微信指数
└── 新榜/飞瓜等数据平台
```

### 5.2 自动化抓取提示

使用 Hermes cronjob + curl/API 轮询热搜，触发阈值时通知。典型 cron job 模式：

```
# 每 30 分钟抓取一次微博热搜
cronjob(action='create', name='weibo-hot-monitor', schedule='30m', ...)
```

发现匹配品牌的热点 → 触发评估流程 → 自动推送提醒。

## 6. 追热点的执行节奏

```
热点出现 (T0)
    │
    ├─ T+15min: 完成匹配度评估（5维度打分）
    │
    ├─ T+30min: 决策（追/不追/观望）+ 产出内容Brief
    │
    ├─ T+60min: 内容初稿完成
    │           ├── 文案：3-5版
    │           ├── 配图/视频：1-2版
    │           └── 品牌安全审核（红线检查）
    │
    ├─ T+90min: 内容发布
    │           ├── 主阵地：微博/抖音
    │           └── 次阵地：小红书/朋友圈
    │
    ├─ T+3h:   数据回收，判断是否追加资源
    │
    └─ T+24h:  结案记录（数据+经验教训→知识库）
```

## 7. 品牌安全审查清单（每次追热点前必过）

- [ ] 是否涉及红线事件（灾难/政治/伤痛）？
- [ ] 热点情绪基调是否与品牌调性冲突？
- [ ] 内容是否存在性别/地域/群体歧视风险？
- [ ] 是否涉及未授权的人物肖像/IP/音乐版权？
- [ ] 如果热点明天反转，品牌会受伤吗？
- [ ] 竞品已经参与了吗？我们是在跟风还是被信息污染？
- [ ] 这条内容会在 3 年后被截图嘲笑吗？（「互联网是有记忆的」）

## 8. 热点类型和策略适配

| 热点类型 | 典型特征 | 最佳策略 | 酒水行业示例 |
|----------|----------|----------|------------|
| **节日型** | 可预见（春节/中秋/世界杯决赛） | 提前 1-2 周准备 | 年夜饭配酒、看球必备 |
| **娱乐型** | 影视/综艺/明星 | 快速轻量，关联场景 | 追剧同款微醺 |
| **现象级 meme** | 网络梗、流行语 | 判断寿命，短期借力 | 「酱香拿铁」式出圈 |
| **体育赛事** | 可预见，情绪高涨 | 场景关联 + 社区激活 | 世界杯看球套餐 |
| **社会议题** | 敏感，需要判断 | **极其谨慎**，大多数建议放弃 | 基本不建议 |
| **天气/季节** | 可预见 | 场景提醒 | 冬日热红酒、夏日冰啤 |
| **行业事件** | 新品发布/行业会展 | 专业角度切入 | 糖酒会热点 |

---

## 9. 反面教材库（追热点翻车常见模式）

1. **灾难消费型**：「XX灾难，我们同在」+ 产品图 → 被骂上热搜
2. **生拉硬扯型**：热点和品牌毫无关联，硬凑 → 品牌被嘲笑「戏多」
3. **迟钝型**：热点已过 48 小时才发 → 被评论「刚通网？」
4. **一次性复制型**：把别家品牌的创意直接替换成自己的 logo → 抄袭骂名
5. **过度营销型**：每个热点都追 → 品牌形象廉价化
6. **反转打脸型**：在定论前站队 → 事件反转后品牌一起翻车

## 危机预备框架

危机公关的实时回应必须由人类决策——Agent 的红线。但危机发生**之前**的准备工作，Agent 可以而且应该做。

### 危机前：风险审计

| 风险维度 | 酒水行业具体风险 | 可能性 | 杀伤力 |
|----------|-----------------|:--:|:--:|
| **产品质量** | 批次异味/沉淀、假酒流入市场、食品安全事件 | 中 | 极高 |
| **营销翻车** | 追热点不当、代言人暴雷、广告涉歧视/敏感 | 中高 | 高 |
| **合规风险** | 广告法违规、未成年饮酒画面、虚假宣传 | 中 | 中高 |
| **供应链** | 经销商窜货致区域价格战升级为公共事件 | 高 | 中 |
| **企业文化** | 员工不当言论、劳资纠纷被曝光 | 低 | 中 |
| **行业黑天鹅** | 政策突变（如禁酒令传闻）、行业丑闻波及 | 低 | 极高 |

### 危机分级

| 级别 | 定义 | 决策人 | 响应时间 | 示例 |
|:--:|------|:--:|:--:|------|
| **L5** | 涉及人身安全/法律诉讼/品牌存亡 | CEO+法务 | <1h | 食品安全事件、假酒致伤 |
| **L4** | 高传播负面、品牌严重受损 | 品牌VP+法务 | <2h | 营销翻车被全网批评 |
| **L3** | 局部负面、可控传播 | 品牌总监 | <4h | KOL 负面评价、局部投诉 |
| **L2** | 轻微负面、小范围 | 品牌经理 | <24h | 个别消费者投诉 |
| **L1** | 潜在风险信号 | 监测+记录 | 持续监控 | 行业趋势预警 |

### 危机团队（RACI）

| 角色 | 职责 | 谁 |
|------|------|:--:|
| **决策人** | 最终拍板——发不发声明、发什么声明 | 按级别定 |
| **对外发言人** | 唯一对外窗口，所有媒体/公众沟通经此人 | 一人，不可多人 |
| **法务审核** | 所有对外声明必经法务审核 | 法务负责人 |
| **信息搜集** | 实时监控舆情、核实事实 | PR 团队 |
| **内部沟通** | 对内同步信息——员工、经销商、投资人 | HR/销售/IR |

### 预备声明模板（Holding Statement）

危机发生后的第一份声明——在事实核实完成之前。不承认、不否认、不猜测。只表达「我们知道、我们在查、我们很快给答案」。

```
【预备声明模板】

我们注意到 [事件描述，客观不带情绪]。
我们正在 [正在做的动作：调查/核实/联系相关方]。
我们将在 [时间承诺：如 24 小时内] 给出进一步说明。
在此期间的任何猜测和传言，请以我们的官方渠道为准。

[品牌名]
[日期/时间]
```

### 危机演练

| 频率 | 内容 | 参与者 |
|:--:|------|------|
| 每年一次 | 模拟 L4/L5 危机：从触发→决策→声明→复盘全流程 | 危机团队全员 |
| 每季度 | 桌面推演：给一个假设场景，30 分钟出预案 | PR + 法务 |

## 品牌安全自动审查

品牌不可逆伤害的成本远高于任何 Campaign 的 ROI。以下规则用于内容产出后的自动扫描——不是替代人工审核，是在人工审核之前拦截明显的雷区。

### 酒水行业敏感词库（自动拦截）

以下词汇出现时，Agent 自动标记并提示人工复核。不自动拦截（上下文可能无害），但必须标注。

**法律/合规类**（高风险——建议拦截）：
- 「未成年人」「学生」「校园」「儿童」+ 任何饮酒/酒类暗示
- 「酒驾」「开车喝酒」「喝了再开」
- 「保健」「疗效」「治疗」「药用」「养生功效」
- 「无限畅饮」「不醉不归」「喝到天亮」——鼓励过量饮酒
- 「女士喝了美容」「男人喝了补肾」——暗示保健功效（广告法禁止）

**品牌安全类**（中风险——强制提示）：
- 灾难/事故词汇：「地震」「洪水」「坠机」+ 品牌名（消费灾难）
- 政治敏感：「国酒」「国宴用酒」（未经授权不可使用）
- 性别歧视暗示：「女人就该喝甜的」「真男人喝烈酒」
- 地域歧视暗示：「XX地方的人就喝这个」
- 竞品名/竞品明显暗示（避免通过贬低竞品给自己惹麻烦）

**竞品相关类**（提醒级——标注但不拦截）：
- 提到竞品品牌名 → 提示「你在提及竞品——确定这是策略需要吗？」

### 社媒内容自动审查规则

内容产出（模式 G）结束后，自动执行以下检查：

| 检查项 | 规则 | 不通过时 |
|--------|------|----------|
| 理性饮酒提示 | 任何涉及饮酒场景的内容→检查是否含「理性饮酒」「适度饮酒」「酒后不驾车」中至少一项 | 自动追加提示语 |
| 未成年暗示 | 内容中出现「年轻」「青少年」「学生」等词 + 饮酒场景 | 标记——人工必须审核 |
| 虚假宣传 | 「最好」「第一」「唯一」等绝对化用语（除非有认证支撑） | 标记——替换为可验证的表述 |
| 价格误导 | 「原价 999 现价 99」类表述——广告法禁止虚构原价 | 拦截——修改 |
| 肖像/版权 | 提及具体人名、品牌名、影视作品名 | 标记——确认授权 |

### 利益相关方风险评估

内容产出后，快速扫三个利益相关方视角：

| 利益相关方 | 如果 TA 看到这个内容 | 检查 |
|------------|---------------------|------|
| **消费者** | 会觉得被冒犯吗？（性别/地域/年龄） | 敏感情感词扫描 |
| **经销商** | 会削弱他们的利润信号吗？（暗示线上比线下便宜？） | 价格信息对比 |
| **监管** | 会被举报吗？（广告法/酒类管理条例） | 合规关键词扫描 |

### 自动审查执行时机

- 模式 G（内容生成）：生成完成后自动触发，输出「审查结果：通过/有条件通过——以下 X 项需人工确认/不通过」
- 模式 F（活动策划）：方案产出后自动触发风险维度扫描
- 模式 A（策略推演）：最终建议输出前触发品牌一致性检查
- Cron job：每日对所有待发内容跑一次增量审查
